    • nginx全局配置指令
    全局配置指令 说明
    user 配置worker进程的主组
    worker_process 指定worker进程数量，一般和cpu核心数相等。有直接参数：auto
    pid 设置pid文件
    worker_connections 设置一个worker进程能够接受并发连接的最大数，
    error_log 所有错误日志写入的文件，如果其它区块中没有error_log出现，那么这个文件会记录所有错误日志。
    include 包含子配置文件
    HTTP的server部分
    http客户端指令 说明
    client_body_buffer_size 防止临时文件写入磁盘，可以通过这个指令为客户端请求体设置缓存大小，默认两个内存页。
    client_body_in_file_only 用于调试或者其它作用，设置为on时能够将请求体强制写入磁盘文件，路径为： client_body_temp_path 指定的。
    client_body_temp_path 定义一个路径用于保存客户端请求体。
    client_body_in_single_buffer 为了减少复制，强制nginx将客户端请求体保存在单个缓存。
    client_body_timeout 读取客户端body的超时时间，会返回408。
    client_header_buffer_size 为客户端请求头指定一个缓存大小，一般不用动，超过1kb时需要使用这个指令
    client_header_timeout 读取客户端请求头的时间长度，在这个时间内没有读取到完整的header部分则会超时并返回408错误。
    client_max_body_size 客户端请求允许的最大的body大小
    keepalive_timeout 指定keepalive持续时间
    large_client_header_buffers 定义最大数量和最大客户端请求头大小。如果client_header_buffer_size不能满足，则会根据这个指令的参数创建缓冲区
    keepalive_requests 该参数限制一个keepalive长连接能够处理的请求数。当连接处理的请求数超过该指令设置的值之后就会关闭该长连接。
    http文件I/O指令
    指令 说明
    open_file_cache 配置一个缓存用于存储打开的文件描述符(打开一个文件的fd之后不进行关闭，将其缓存起来，下次请求这个文件的时候直接使用这个fd，避免open、close等系统调用操作)、目录查询、文件查询错误(例如一个文件不存在，nginx会查询文件是否存在，不存在则返回客户端404，此功能能将这些错误信息也缓存起来，下次直接返回错误，而不用去尝试读取文件了)。open_file_cache max=1000 inactive=60s; max：设置缓存中的最大元素个数，inactive：定义一个时间，在此期间为访问的元素会从该缓存中删除，默认60s。
    open_file_cache_errors 对于一些文件访问错误的信息是否缓存，默认off。
    open_file_cache_min_uses open_file_cache指令中的inactive参数时间内文件的最少使用次数，如果超过这个数字，文件fd一直是在缓存中保留的。
    open_file_cache_valid 对open_file_cache缓存有效性检查的时间间隔，发生变化则更新。如果是常改动的文件则需要将次指令的值调小点，对于不常改动的文件则可以调大点。
    http hash指令
    指令 说明
    server_names_hash_bucket_size 指定用于保存server_name散列表大小的桶
    server_names_hash_max_size 指定server_name散列表最大大小
    types_hash_max_size 用于存储散列表的桶的大小
    types_hash_max_size 指定散列类型表的最大大小
    variables_hash_bucket_size 指定用于存储保留变量桶的大小
    variables_hash_max_size 指定存储保留变量最大散列值的大小
    http socket指令
    指令 说明
    lingering_close
    lingering_time
    lingering_timeout
    reset_timeout_connection 使用这个指令后，超时的客户端链接将会被立即关闭，释放相关内存。默认的状态是出于FIN_WAIT1，这种状态将会一直保持连接。原理：发送rst报文快速断开连接回收内存和端口，避免断开进入timewait状态，
    send_lowat
    send_timeout 在两次的客户端接受响应的写操作之间设置一个超时时间。
    tcp_nodelay off启用或者on禁用tcp nagle算法
    tcp_nopush 仅依赖于sendfile的使用，使nginx在一个数据包中尝试发送响应头以及在数据包中发送一个完整的文件。

    虚拟主机部分

    Listen指令的参数
    指令 说明
    default_server 请求进来通过host匹配虚拟主机，没有匹配上的会交给这个server处理。
    backlog 指定nginx当前监听的套接字半连接队列的长度。半连接队列：tcp中的概念，指没有完成三次握手的半连接队列
    ssl 表明该端口仅仅接受https的连接
    so_keepalive 该参数为tcp监听套接字配置keepalive
    server_name

    通配符：*.example.com www.example.\* .example.com(特殊形式，匹配*.example.com也包括example.com)。在域名前加波浪号~，正则表达式也可以被作为参数应用于server_name。

    Location
    修饰符 处理方式
    = 使用精确匹配并终止搜索
    ~ 区分大小写的正则表达式匹配
    ~* 不分区大小写的正则表达式匹配
    ^~ 如果该location是最佳的匹配，那么对于匹配这个location的字符串不再进行正则表达式检测。注意这不是一个正则表达式匹配—它的目的是优先于正则表达式的匹配

    没有正则的location匹配优先级高于带正则匹配的location

    指令 说明
    alias alias替代location中匹配的部分
    internal
    limit_except 限定一个location可以执行的http操作，limit_except POST {deny all}，仅仅允许post方法请求。

    带有前缀“=”的location 或者 命名location不可以嵌套location。

    使用文件做缓存

    使用expires设置文件的缓存时间，原理就是修改http协议中的header：Cache-Control

    server {
    listen 80;
    server_name abc.com;
    root /data/www;
    location ~* /.*\.(css|js)$ {
        expires 5m;
    }
    location ~* /.*\.(png|jpg|jepg)$ {
        expires 1d;
    }
    access_log /usr/local/nginx/logs/cache.log main;
}

    代理模块

    proxy_pass

    反向代理，该指令有一个参数，URL请求将会被转换，带有URL部分的proxy_pass指令将会使用该URL代替request_uri部分。

    listen 80;
server_name www.abc.com;
location /uri {
    proxy_pass http://localhost:8080/newuri;
}
# 访问/uri，当请求传递到上游服务器时会被替换为/newuri，如果proxy_pass指令不带url，则传递到上游服务器时仍然是/uri路径。


location ~ ^/local{
    proxy_pass http://localhost:8080/foreign;
}
# 当location使用了正则表达式时，那么在uri部分不会发生转换，请求传递到上游服务器时uri仍然是/local而不是/foreign，高版本的nginx这么写配置文件语法检测无法通过

location / {
    rewrite /(.*)$ /index.php?page=$1 break;
    proxy_pass http://localhost:8080/index;
}
# 如果location内有rewrite规则改变了url，那么nginx使用这个uri处理请求，不再发生转换。
# 例如这个例子，请求到达上游服务器时将会是/index.php?page=，而不是预期的/index
    proxy_connect_timeout

    与上游服务器(即：proxy_pass http://xxxx.xxxx.com)连接超时的时间

    proxy_hide_header

    拦截上游服务器返回给客户端的header。

    例：proxy_hide_header ResponseHeader1;。

    设置之后如果上游服务器在返回给客户端的response中设置了请求头ResponseHeader1，会被nginx拦截掉，客户端无法收到。

    proxy_http_version

    设置nginx与上游服务器通信的http协议版本。

    proxy_ignore_headers

    处理上游服务器的响应时，忽略proxy_ignore_headers指令指定的header。不对指定的header进行处理。proxy_hider_header是不将上游服务器的header响应给客户端，proxy_ignore_headers是不处理上游服务器响应的header。

    proxy_pass_header

    该指令覆盖掉proxy_hide_header指令设置的header，允许header返回给客户端。

    例如：proxy_pass_header ResponseHeader3;

    proxy_pass_request_body

    如果设置为off，则会阻止请求体(body)发送给上游服务器。经测试：如果请求不设置body(body为空)则请求能正常到达上游服务器，如果设置了body则请求会被阻塞。

    proxy_pass_request_headers

    如果设置为off，则会阻止客户端的header发送给上游服务器，上游服务器无法收到来自客户端请求的header，但是在nginx通过proxy_set_header设置的请求头依然能传递给上游服务器。

    proxy_read_timeout

    从上游服务器读取响应的超时时间，如果上游服务器处理较慢则应该设置的偏大一点。

    proxy_send_timeout

    对上游服务器进行写操作的超时时间。

    proxy_set_body

    修改来自客户端请求的body的内容。

    proxy_set_header

    修改客户端请求中的header值，如果设置为空，则不发送给上有服务器。

    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header name “”;

    upstream模块

    ip_hash
    keepalive
    least_conn
    server
    1. weight：权重，权重越高转发的请求比例越高。
    2. fail_timeout：在这个响应时间内，上游服务器必须做出响应，如果在这个时间内无响应且超过了max_fails次数则该server会被标记为down。
    3. max_fails：与上游服务器建联的最大失败次数。
    4. backup：一旦其它服务器宕机，那么只有标记为backup的server才会接受请求。
    5. down：标记一个server，标记之后该server不再接受请求。
    6. keepalive：指定每一个worker进程缓存到上游服务器的连接数。
    
upstream test {
    ip_hash;
    keep_alive 32; # 如果在高峰期需要多于32个连接，那么nginx会打开更多的连接以满足需要，在空闲时关闭最少使用的连接，是这个数值回到32
    server 192.168.0.1;
    server 192.168.0.2;
}

location / {
    proxy_http_version 1.1;
    proxy_set_header Connection ""; 此header的含义是当client和server通信时对于长链接如何进行处理。
    proxy_pass http://test;
}

    server指令

    server_name

    server{
    listen 80 default_server;
    server_name _;
    return 444; # nginx接受到的请求中，如果host没有匹配上相应的虚拟主机，则服务器这边主动断开连接，不发送任何响应
}

    server_tokens

    server{
    listen 80;
    server_tokens off; # 禁止发送nginx的版本号
    server_name www.abc.com;
    index index.html;
    root /data/www/;
}

    log_format fff '$http_user_agent, $remote_addr, $status';

    log_not_found 设置为off时，禁止在错误日志中报告404错误，默认值为on
